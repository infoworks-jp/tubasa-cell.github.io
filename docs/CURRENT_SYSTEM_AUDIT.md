## 監査方針

既存機能・既存データを削除または置換せず、確認できた事実と未確認事項を分けて記録する。

- 給与、仕入・外注支払、税・社会保険等を分類し、主要負担を表示する処理がある。
- GitHub Actionsを使った更新処理が導入されている。
- 2026年7月22日の券売機データ反映用コミットと、その抽出エラー修正コミットが存在する。

## 現時点の重大な構造リスク

1. `index.html` 一枚に画面、計算、データが集中している。
2. 日付追加のたびに巨大な埋込データを書き換えるため、抽出・置換失敗が起きやすい。
3. 更新処理が正規表現や文字列置換に依存すると、軽微なレイアウト変更でも失敗する。
4. 読み込み、検証、保存、集計、公開の各工程が分離されていない。
5. 原本と変換後データの対応関係、取込履歴、重複判定、修正履歴が十分に見えない。
6. 公開画面と管理用データの境界が弱く、今後の個人情報・請求書運用には不向き。

## PDF請求書読み込みについて

PDF請求書読み込みは既存機能として扱い、将来機能へ降格させない。

ただし、現時点でGitHub上のどのファイルがPDF解析本体か、検索結果だけでは特定できていない。削除・再実装は行わず、履歴、ワークフロー、スクリプト、管理画面を追加調査して実装位置を確定する。

## 完成と呼ばない条件

以下のいずれかが未確認なら完成扱いにしない。

- 原本を投入して取り込める
- 同じ原本を二度投入しても重複登録されない
- 読取結果を人が確認・訂正できる
- 売上、入金、仕入、給与、社会保険、固定費が同一期間で検算される
- 公開画面に個人情報や原本が露出しない
- GitHub Pagesで実際に表示される
- Mac、Windows、スマートフォンで主要操作を確認できる
- 更新後に旧データが欠落していない
